- This is a repo that determines all the system configuration. For example if you are wanting to modify the system configuration for neovim, go ahead and look in "~/limonene/nicole/common/nvim.nix" not "~/.config/nvim". Simlarly window management is in "~/limonene/home/wm/sway.nix".
- This pattern uses the dendritic pattern for managing its nix modules (more info: https://raw.githubusercontent.com/mightyiam/dendritic/refs/heads/master/README.md) but for a quick TLDR: 
> In the dendritic pattern every Nix file except for entry points such as default.nix and flake.nix is a module of the top-level configuration. In other words, every Nix file that isn't an entry point is a Nixpkgs module system module that is imported directly into the evaluation of the top-level configuration. Additionally, every top-level module implements a single feature across all configurations that that feature applies to and is at a path that serves to name that feature.

- This also handles configurations for macos and linux simultaneously the folder "nicole/common" is for utilities that exist in both mac and linux, and thd folder "nicole/linux" is for linux specific packages.
- Oftentimes if you need to look up nixos options, I would highly recommend sites like "https://mynixos.com/home-manager/options/programs.firefox" or "https://mynixos.com/nixpkgs/options/services.jellyfin" that allow you to look up the specific options
- Its often a good idea to run the system configuration in a dry run using: "nix build --dry-run ~/limonene#nixosConfigurations.$(hostname).config.system.build.toplevel" to check for any errors, and validate that the system configuration changes arent going to do something computationally infeasible like rebuilding chromium/linux from scratch.
